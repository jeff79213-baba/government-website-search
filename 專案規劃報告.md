# 政府機關網站找答案痛點處理 — 專案規劃報告

## 一、問題定義

臺灣政府機關網站分散，民眾在搜尋特定服務（如房屋稅率查詢、勞保給付、戶政事務等）時，常遇到以下困難：

1. **搜尋結果混雜**：Google 搜尋出現大量廣告、非官方資訊，難以找到正確的政府官網
2. **網址驗證困難**：即使找到疑似官網，仍需人工一一點擊確認是否為正確網站
3. **各縣市差異**：許多服務（如房屋稅）各縣市有不同官網，民眾不易找到自己所在地的對應網站
4. **缺少社群回饋機制**：無法知道某個網址是否正確、是否已失效

## 二、目標

建立一個**群眾協作的政府網站導航工具**，讓使用者：

- 輸入關鍵字，快速找到正確的政府機關網址
- 透過按讚/倒讚、留言回饋網址正確性
- 提交新網址（經管理員審核後上架）
- 查看網址使用率（%）作為選擇參考

## 三、核心功能

| 功能 | 說明 |
|------|------|
| 關鍵字搜尋 | 比對網站名稱、描述、分類、標籤 |
| 分類過濾 | 依戶政、稅務、勞保、健保、教育等分類篩選 |
| 縣市引導 | 若該服務因縣市而異，先引導使用者選擇縣市，再顯示對應網址 |
| 按讚 / 倒讚 | 讓使用者評價網址的正確性與實用性 |
| 留言功能 | 使用者可回報「網址正確」、「已失效」等資訊 |
| 提交新網址 | 使用者可貢獻新發現的政府網站（待管理員人工審核） |
| 使用率統計 | 統計每個網址被點擊的次數與佔比 |
| 資料匯出 | 可匯出全部資料（JSON / CSV），確保資料可攜性 |

## 四、技術架構

```
使用者（手機 / 電腦）
    ↑
PWA 網頁（漸進式網頁應用）
    ↑ 可加入手機主畫面，也可包裝上架 App Store / Google Play
Supabase（免費後端方案）
    ├── PostgreSQL 資料庫（存放網址、按讚、留言等）
    ├── API 接口
    └── 可匯出全部資料
```

### 技術選擇說明

| 層級 | 選擇 | 原因 |
|------|------|------|
| 前端 | PWA（漸進式網頁應用） | 一套程式碼同時滿足「網頁」與「App」需求 |
| 後端 | Supabase（免費方案） | 零成本、內建資料庫與 API、支援資料匯出 |
| 部署 | Vercel 或 Netlify | 免費託管網頁 |
| App 上架 | PWABuilder / Bubblewrap | 將 PWA 包裝成 iOS / Android App |

### 資料安全性評估

- 儲存資料：政府公開網址、使用者評價（按讚/留言）
- 不涉及：個資（身份證字號、地址、電話等）
- 可攜性：Supabase 支援完整資料匯出（JSON / CSV），隨時可搬家

## 五、網址審核機制

**核心原則：網址正確性以人工審核為主，社群回報為輔。**

- 管理員（你）手動確認每個網址是否為正確的政府官網
- 使用者可透過按讚/倒讚、留言回饋網址狀態
- 使用者可提交新網址，由管理員審核後上架

## 六、使用流程示例（以房屋稅為例）

```
使用者輸入「房屋稅率查詢」
  → 系統提示選擇縣市（臺北市 / 新北市 / 高雄市 …）
    → 使用者選擇「臺北市」
      → 顯示：
          臺北市稅捐稽徵處
          https://tpctax.gov.taipei/
          👍 95%  |  💬 12 則留言
```
